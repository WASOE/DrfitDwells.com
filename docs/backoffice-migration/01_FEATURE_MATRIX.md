## Batch 8B-read — OPS cabin detail read parity (2026)

- **`GET /api/ops/cabins/:id`** (`cabinsReadModel.getCabinDetailReadModel`): additive fields only — e.g. `geoLocation`, `hostName`, `experiences`, `transportCutoffs`, `pricingModel`, `minGuests`, `blockedDatesCount` (single), `unitBlockedDatesSummary` (multi), without removing or renaming existing keys.
- **`/ops/cabins/:id`**: read-only sections for location/geo, meeting point & arrival, safety & emergency, transport (options + cutoffs), highlights/badges/experiences, legacy blocked-date summary, packing list; no new writes.
- **Cabin PATCH/create** and admin surfaces unchanged; goal is a better OPS single pane before moving advanced writes.
