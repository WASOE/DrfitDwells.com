# The Cabin — arrival route files

Route geometry, KML, GPX, and GeoJSON are generated by:

`npm run guides:export-the-cabin`

The public guide **does not embed an in-browser map** until a real offline raster basemap is approved. Guests use **GPX/KML in a map app** from this folder.

## Ops checklist: raster `arrival-basemap.pmtiles`

Use this when you are ready to make the **in-page** MapLibre view match real terrain/roads (guests should still rely on **GPX/KML in a map app** first).

1. **Source** — Use a **licensed or owned** raster export (satellite or topo) for the **Bachevo → The Cabin** corridor only. Keep the bbox tight to limit file size.
2. **Format** — Produce a **raster** PMTiles archive (PNG / JPEG / WebP tiles inside PMTiles). **Vector** PMTiles are not wired to a style in this guide.
3. **Filename** — Place the file here as: `client/public/guides/the-cabin/arrival-basemap.pmtiles` (served at `/guides/the-cabin/arrival-basemap.pmtiles`).
4. **Verify** — After wiring an in-page map again, confirm the basemap loads, pan/zoom, and check that the recommended route and pins align with the imagery.
5. **Ship** — Commit the PMTiles with the repo (or your asset pipeline) so production and offline caching stay self-hosted.
6. **Copy** — You may still want a one-line note on the guide that **offline navigation = GPX/KML in OsmAnd/Organic Maps**; the basemap improves **in-browser context**, not replacing app import.

## Technical notes

- Expected filename: `arrival-basemap.pmtiles`
- **Vector** PMTiles are not auto-styled by this guide (use raster for the bundled basemap path).

Tools that can produce small regional PMTiles include [Protomaps / PMTiles](https://github.com/protomaps/go-pmtiles) and workflows built on `gdal2tiles` / `tilemaker` + conversion to PMTiles.

Until an in-page map ships, **GPX/KML + map apps** are the only guided map experience on the web page.
